# 📌 02 - MongoDB vs RDBMS

## 1️⃣ What is the Difference Between MongoDB and RDBMS?

MongoDB is a **NoSQL document-oriented database**  
RDBMS (Relational Database Management System) is a **table-based relational database**

Examples of RDBMS:
- MySQL
- PostgreSQL
- Oracle
- SQL Server

MongoDB stores data as **documents**, while RDBMS stores data in **tables (rows & columns)**.

---

## 2️⃣ Structural Difference

### 🔹 RDBMS Structure

Database → Tables → Rows → Columns

Example (Students Table):

| id | name  | age | cgpa |
|----|-------|-----|------|
| 1  | Bhanu | 23  | 8.5  |
| 2  | Anil  | 24  | 8.9  |

Schema must be predefined:
```sql
CREATE TABLE students (
  id INT PRIMARY KEY,
  name VARCHAR(50),
  age INT,
  cgpa DOUBLE
);
```

---

### 🔹 MongoDB Structure

Database → Collections → Documents → Fields

Example document:

```json
{
  "name": "Bhanu",
  "age": 23,
  "cgpa": 8.5
}
```

No fixed schema required.

---

## 3️⃣ Detailed Comparison Table

| Feature | MongoDB | RDBMS |
|----------|----------|----------|
| Data Storage | Documents (JSON/BSON) | Tables |
| Schema | Flexible | Fixed |
| Joins | Limited | Strong JOIN support |
| Scaling | Horizontal (Sharding) | Vertical scaling |
| Transactions | Supported (but newer) | Strong ACID |
| Query Language | Mongo Query Language (MQL) | SQL |
| Best For | Big Data, Microservices | Banking, ERP systems |

---

## 4️⃣ Students Example Comparison

### 🔹 In RDBMS

To insert:

```sql
INSERT INTO students (name, age, cgpa)
VALUES ('Bhanu', 23, 8.5);
```

Schema must match exactly.

---

### 🔹 In MongoDB

```javascript
db.students.insertOne({
  name: "Bhanu",
  age: 23,
  cgpa: 8.5
});
```

You can also insert:

```javascript
db.students.insertOne({
  name: "Anil",
  age: 24,
  cgpa: 8.9,
  internship: true
});
```

Extra field allowed.

---

## 5️⃣ Why This Matters in Production (SRE Perspective)

### Scenario 1: Rapid Application Change

Developer adds new field:
`department`

In RDBMS:
- ALTER TABLE required
- Downtime risk
- Migration script needed

In MongoDB:
- Just start inserting new field
- No schema change required
- Zero downtime

---

### Scenario 2: Performance Issue

In RDBMS:
- Heavy JOIN between 5 tables → slow query

In MongoDB:
- Data often embedded in single document
- Faster reads

As SRE, understanding this helps in:
- Root cause analysis
- Capacity planning
- Performance debugging

---

## 6️⃣ When to Use MongoDB vs RDBMS

### Use MongoDB When:
- Schema changes frequently
- Microservices architecture
- High read/write throughput
- Unstructured or semi-structured data

### Use RDBMS When:
- Strong relationships required
- Complex JOIN operations
- Strict ACID compliance needed
- Banking / Financial systems

---

## 7️⃣ Common Production Issues Based on Choice

### ❌ Issue: Trying to use MongoDB like RDBMS

Developers:
- Create too many collections
- Avoid embedding
- Perform unnecessary lookups

Result:
- Slow queries
- Increased CPU usage

---

### ❌ Issue: Large Documents

If embedding too much data:
- Document size exceeds 16MB limit
- Insert fails

## 🧠 SRE Key Takeaway

MongoDB is optimized for:
- Flexibility
- Scalability
- Fast development

RDBMS is optimized for:
- Data integrity
- Strong relationships
- Structured systems

As SRE:
You must understand **why the application chose MongoDB**, because production troubleshooting depends on it.
